## 🎯 使用指南

這份檢查清單基於AIOGEO實施指南設計，幫助您系統性地優化內容以提升AI引用潛力。建議您：

✅ **逐項檢查**：每篇內容都要通過所有檢查點  
✅ **定期審核**：每月重新檢視已發布內容  
✅ **持續改進**：根據AI引用效果調整策略  
✅ **團隊共享**：讓內容團隊都使用統一標準

### 📊 評分標準
- **A級 (90-100分)**：AI引用潛力極高
- **B級 (80-89分)**：引用潛力良好  
- **C級 (70-79分)**：需要優化改進
- **D級 (60-69分)**：急需重大調整
- **F級 (60分以下)**：不符合GEO標準

## 🏗️ 一、內容結構優化 (25分)

### 📝 基礎結構檢查

**□ 1.1 文章摘要 (5分)**
- [ ] 開頭包含150-300字的summary段落
- [ ] 摘要清楚說明文章主要價值
- [ ] 使用`<section class="summary">`標籤包裝
- [ ] 摘要能獨立回答用戶核心問題

**□ 1.2 標題層級 (5分)**
- [ ] 使用清晰的H1-H4標題結構
- [ ] 每個標題都具有語義價值
- [ ] 標題間保持邏輯層級關係
- [ ] 避免過度嵌套（不超過H4）

**□ 1.3 段落組織 (5分)**  
- [ ] 每段落聚焦單一概念或主題
- [ ] 段落長度適中（3-6句話）
- [ ] 使用過渡詞連接段落關係
- [ ] 重要資訊前置表達

**□ 1.4 內容完整性 (5分)**
- [ ] 回答用戶可能的所有相關問題
- [ ] 提供actionable的具體建議
- [ ] 包含實際案例或範例
- [ ] 結論總結關鍵要點

**□ 1.5 可讀性優化 (5分)**
- [ ] 使用項目符號和編號清單
- [ ] 重要資訊用加粗或框架突出
- [ ] 適當使用表格整理複雜資訊
- [ ] 插入相關圖片或圖表輔助說明

---

## ⚙️ 二、技術實作標準 (25分)

### 🏷️ 結構化資料標記

**□ 2.1 Article Schema (8分)**
```json
必須包含的基礎資料：
- headline: 文章標題
- author: 作者資訊
- datePublished: 發布日期
- description: 文章描述
- publisher: 發布機構
- mainEntityOfPage: 頁面URL
```

**檢查項目**：
- [ ] 加入完整的Article Schema標記
- [ ] 所有必要欄位都已填寫
- [ ] 使用JSON-LD格式（推薦）
- [ ] 通過Google Schema檢測工具驗證

**□ 2.2 FAQ Schema (5分)**
- [ ] 將內容中的問答提取為FAQ Schema
- [ ] 每個問題都有明確完整的答案
- [ ] 使用Question和Answer類型標記
- [ ] FAQ內容與文章內容一致

**□ 2.3 語意HTML標籤 (4分)**
- [ ] 使用`<article>`包裝完整文章
- [ ] 用`<section>`區分不同主題段落
- [ ] 用`<header>`包含標題和meta資訊
- [ ] 正確使用`<time>`標記日期

**□ 2.4 Meta資料優化 (4分)**
- [ ] 設定descriptive的title標籤
- [ ] 撰寫吸引人的meta description
- [ ] 加入適當的keywords標籤
- [ ] 設定Open Graph社群分享資料

**□ 2.5 連結結構 (4分)**
- [ ] 內部連結使用描述性錨點文字
- [ ] 加入適當的外部權威來源連結
- [ ] 設定rel="nofollow"在適當的外部連結
- [ ] 確保所有連結都可正常訪問

## 🔗 三、AI引用潛力 (25分)

### 💡 可引用性設計

**□ 3.1 權威性建立 (8分)**
- [ ] 引用至少2個權威來源或研究
- [ ] 提供具體的統計數據或案例
- [ ] 標明作者專業背景和可信度
- [ ] 加入相關證書或認證資訊

**□ 3.2 事實準確性 (6分)**
- [ ] 所有數據都有明確來源標註
- [ ] 避免主觀判斷，以客觀事實為主
- [ ] 定期更新過時資訊
- [ ] 更正已發現的任何錯誤

**□ 3.3 問答模組化 (6分)**
- [ ] 包含至少3個直接問答對
- [ ] 問題涵蓋用戶常見疑惑
- [ ] 答案簡潔明確，易於AI擷取
- [ ] 使用`<details>`和`<summary>`標籤

**□ 3.4 引用友好格式 (5分)**
- [ ] 重要概念用定義框或引用塊突出
- [ ] 建立「關鍵要點」或「總結」段落
- [ ] 使用數字清單呈現步驟或要點
- [ ] 提供可獨立理解的段落內容

- [ ] 提供圖表的文字版本說明

- [ ] 資源包含完整的使用指南
- [ ] 加入明確的下載連結和說明

**□ 4.4 影音內容 (6分)**

- [ ] 加入章節標記和時間戳記
