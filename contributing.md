## 📋 貢獻流程

### 🚀 快速開始

1. **Fork 專案**
   ```bash
   # 前往 GitHub 點擊 Fork 按鈕
   https://github.com/bless25min/AIOGEO-Knowledge
   ```

2. **Clone 到本地**
   ```bash
   git clone https://github.com/你的用戶名/AIOGEO-Knowledge.git
   cd AIOGEO-Knowledge
   ```

3. **建立新分支**
   ```bash
   git checkout -b feature/你的功能名稱
   # 或
   git checkout -b content/新文章主題
   ```

4. **進行修改**
   - 新增或修改內容
   - 確保符合品質標準
   - 測試修改效果

5. **提交變更**
   ```bash
   git add .
   git commit -m "描述你的修改內容"
   git push origin feature/你的功能名稱
   ```

6. **建立 Pull Request**
   - 前往 GitHub 建立 PR
   - 詳細描述你的修改
   - 等待審核和合併

```markdown

---

<section class="summary">

這段內容最有可能被 AI 搜尋引擎直接引用。
</section>
```

#### 內容品質標準

**✅ 必須符合的標準**：
- [ ] 事實準確，有可靠來源支撐
- [ ] 邏輯清晰，結構完整
- [ ] 語言簡潔，易於理解
- [ ] 實用性強，有實際價值
- [ ] 符合 GEO 優化標準

**📊 推薦的內容元素**：
- [ ] 開頭摘要（summary section）
- [ ] 清晰的標題層級（H2, H3, H4）
- [ ] 實用的表格和列表
- [ ] 具體的代碼範例
- [ ] 檢查清單或行動指南
- [ ] FAQ 問答部分

#### 引用和來源規範

**引用格式要求**：
```markdown
<!-- 學術文章引用 -->
根據 Smith et al. (2024) 的研究顯示...

<!-- 網路資源引用 -->
根據 [OpenAI 官方文檔](https://openai.com/docs) 的說明...

<!-- 統計數據引用 -->
市場調查顯示，78% 的用戶偏好使用 AI 搜尋工具 ([Statista, 2024](https://statista.com/ai-search))。
```

**來源品質要求**：
- 優先引用權威機構發布的資料
- 確保所有連結在發布時可正常訪問
- 標註資料的發布時間
- 避免引用明顯有偏見的來源

### 🔧 技術修改指南

#### 代碼提交規範

**提交訊息格式**：
```bash
類型(範圍): 簡短描述

詳細描述（如果需要）

# 類型可以是：
- feat: 新功能
- fix: 修復問題
- docs: 文檔更新
- style: 格式修改
- refactor: 代碼重構
- test: 測試相關
- chore: 雜項修改

# 範例：
feat(search): 增加全文搜尋功能
fix(layout): 修復移動端顯示問題
docs(geo): 更新 GEO 基礎原理文章
```

#### 技術標準

**前端代碼**：
- 遵循 HTML5 語意化標準
- CSS 使用響應式設計
- JavaScript 確保跨瀏覽器相容性
- 優化網站載入性能

**內容格式**：
- Markdown 語法正確
- 結構化資料標記完整
- 圖片包含適當的 alt 屬性
- 連結使用描述性錨點文字

<summary><strong>Q：我不是技術背景，也能貢獻內容嗎？</strong></summary>
<p>當然可以！我們非常歡迎非技術背景的貢獻者。你可以：1) 撰寫或改進文章內容；2) 分享實戰經驗和案例；3) 提供使用者體驗反饋；4) 協助推廣和社群建設。我們提供完整的指導和支援。</p>

<details>
<summary><strong>Q：提交的內容需要多久才會被審核？</strong></summary>

</details>
